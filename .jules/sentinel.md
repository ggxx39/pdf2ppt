## 2024-05-24 - Secure Error Handling\n**Vulnerability:** Information leakage through unhandled external API error messages propagated to the UI.\n**Learning:** Error bounds in services `geminiService.ts` and UI components `Workspace.tsx` exposed raw API outputs containing potential internal states.\n**Prevention:** Catch external API errors, log the actual error internally (console) for debugging, but expose generic/sanitized errors to the UI or calling functions to avoid data leakage.
