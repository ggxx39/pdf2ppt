## 2024-05-11 - Client-Side DoS and Information Leakage Prevention
**Vulnerability:** The application was vulnerable to client-side Denial of Service (DoS) due to unbounded file uploads being read completely into memory via `FileReader`. Additionally, the `catch` block interacting with the Gemini API was logging raw internal error messages directly to the user-facing UI component, which could leak internal pathing, API issues, or stack traces.
**Learning:** In purely client-side architectures like this Vite/React app, processing large files natively in the browser without size limits can quickly exhaust the user's available RAM and crash the tab. Secondarily, even in a client-side app, catching errors and reflecting `error.message` verbatim to the UI risks exposing technical debt and internal configurations to end users.
**Prevention:** Implement strict file size limits (e.g., 10MB) before loading any file into `FileReader` to prevent memory exhaustion. Sanitize all error messages shown in the UI while preserving the raw error only in `console.error` for developer debugging.
