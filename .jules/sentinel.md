## 2023-10-27 - [Path Traversal in File Download via Untrusted Filename]
**Vulnerability:** A critical path traversal vulnerability was found in `Workspace.tsx` where the `downloadPPTX` function concatenated the untrusted `file.name` directly into the save path (`pptx.writeFile({ fileName: \`\${file?.name.split('.')[0]}_Converted.pptx\` })`).
**Learning:** Even client-side file generation libraries (like `pptxgenjs` or standard download anchors) can be susceptible to path traversal if they pass user-controlled filenames containing directory traversal characters (`../`, `\`, `/`) directly to the browser's download API or file system access API, potentially overwriting unintended files depending on browser mitigations. Additionally, leaking raw error messages (like `error.message`) to the UI can expose sensitive information like API keys.
**Prevention:** Always sanitize or strictly validate filenames derived from user input (e.g., `file.name`) by stripping or replacing path traversal characters (e.g., `replace(/[\/\\]/g, '_')`) before using them in file download/save operations. Fail securely by replacing raw error messages with generic, user-friendly equivalents.
