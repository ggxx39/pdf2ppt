## 2025-04-19 - Route-Level Code Splitting for Heavy Dependencies
**Learning:** The architectural intent to dynamically load heavy dependencies like `pptxgenjs` and `recharts` only when needed was completely bypassed because `App.tsx` statically imported the route components (`Dashboard`, `Workspace`, `ConfigEditor`, `LogPanel`). This caused the main bundle to swell to 1.25 MB since Vite statically bundles imported dependencies, even if they're conditionally rendered within JSX based on an `activeTab` state.
**Action:** Always use `React.lazy()` and `Suspense` when conditionally rendering top-level route/tab components in a single-page app (SPA), especially when those components encapsulate large third-party libraries. This ensures true code splitting and dramatically reduces the initial load time and memory footprint.
