## 2024-05-10 - Route-level Code Splitting for Heavy Modules
**Learning:** The application bundles heavy dependencies like `pptxgenjs` (used in `Workspace`) and `recharts` (used in `Dashboard`) together in the initial load, leading to a large main chunk size. Since the application uses conditional rendering based on state (`activeTab`) rather than a dedicated routing library, these modules are parsed and executed upfront even if the user hasn't visited those tabs.
**Action:** Implemented route-level code splitting by dynamically loading tab components using `React.lazy()` and wrapping the render area in a `<Suspense>` boundary. This ensures that the heavy libraries are only fetched when their respective tabs are requested, drastically reducing initial JS execution time and main bundle size.
