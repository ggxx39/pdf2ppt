## 2025-02-28 - Route-Level Code Splitting for Heavy Dependencies
**Learning:** The application was statically importing route components (`Dashboard`, `Workspace`, `ConfigEditor`, `LogPanel`), which pulled in heavy dependencies like `recharts` and `pptxgenjs` into the initial bundle, blocking the main thread and increasing the time to interactive (TTI) for the entire app. This is a common performance bottleneck in React SPAs without a dedicated router library handling this automatically.
**Action:** Implementing route-level code splitting using `React.lazy` and `Suspense` allows these components and their heavy sub-dependencies to be downloaded and evaluated only when the user navigates to their specific tabs. This significantly reduces the initial bundle size. Always evaluate the initial dependency tree and utilize dynamic imports for heavy, route-specific libraries.
